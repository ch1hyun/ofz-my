Code #5

The error is caused by the deletion of the `fileName` object before the `pdf_doc` object is deleted. The `pdf_doc` object still references the `fileName` object when it is deleted, causing a heap-use-after-free error. To fix this, the `fileName` object should be deleted after the `pdf_doc` object is deleted.

Here is the fixed code:
